# AI_assistant 项目规划（按当前代码进度校正版）

> 版本：2026-03-07（按代码现状更新）
> 适用范围：当前仓库（C++20 + SDL3 + ImGui + ONNX Runtime + OpenCV）

---

## 1. 规划目标与更新原则

本文档用于替代旧版“目标先行、进度滞后”的规划描述，改为：

1. **以仓库现状为基线**（已实现能力、真实模块边界、当前技术债）。
2. **按阶段标注状态**（已完成 / 进行中 / 未开始），避免“看起来都完成”。
3. **将接下来 8 周执行项与代码落点绑定**，保证可执行、可验收。

## 2. 当前基线（截至 2026-03-06）

## 2.1 已稳定落地（可运行）

### A. 运行时与生命周期主链路
- SDL3 窗口、渲染、托盘、点击穿透、可见性切换已接通。
- 应用生命周期已拆分为 Init / Bootstrap / Run / Teardown。
- 运行时主循环已分离为事件、Tick、渲染入口。

### B. 感知管线（Perception Pipeline）
- 屏幕采集：DXGI 桌面捕获可用。
- 场景分类：MobileCLIP ONNX 推理链路可用。
- OCR：PP-OCR det + rec 已接入，异步执行、超时降级（保留 last stable）已落地。
- 系统上下文：进程名 / 窗口标题 / URL hint 已接入。
- 摄像头人脸/情绪：OpenCV + ONNX 已接入，含低光/遮挡降级、缺帧插值、平滑处理。

### C. 行为与插件
- 插件生命周期管理已具备（Init / Update / Destroy）。
- 插件工作线程含超时降频、连续失败自动禁用、定时自动恢复。
- 交互行为输出与插件输出可混合后应用到 Runtime。

### D. ASR / Chat 混合路由骨架
- Offline / Cloud / Hybrid Provider 架构已打通。
- Hybrid 具备“低置信触发云端、失败回退离线”策略壳。
- ASR 已接通 VAD 分段与运行时指标统计（RTF、超时率、云调用比等）。

### E. 编辑能力（运行时内嵌）
- 部件选中、拖拽、Gizmo 操作链路可用。
- Undo/Redo、保存 model.json 能力可用。
- 编辑状态已通过 facade/service 拆分，不再全部堆在单文件。

### F. 工程测试
- 插件生命周期与线程隔离测试已存在（plugin_lifecycle_tests）。

## 2.2 已实现但仍属“占位/过渡”

1. **ASR 识别内容仍为占位输出**（离线/云端 provider 未接入真实识别引擎调用）。
2. **Chat 回答仍为占位输出**（offline/cloud provider 返回 placeholder 文本）。
3. **OnnxBehaviorPlugin 目前仍以规则输出为主**，尚未接入真实 onnx 推理图执行路径。
4. **OCR 已具备 det+rec 主链路，但 domain_tags 主要靠规则推断**，语义层面仍是 1.0 版本。

---

## 2.3 与旧规划不一致项（本次已修正）

1. 旧规划中“语音与对话接近可用”描述过于乐观，现调整为：**架构完成、模型能力未完成**。

3. “自主学习闭环”此前写法偏完成态，现明确为：**尚未启动**。

## 4. 各阶段详细规划（重排后）

## Phase A：运行时与工程底座（已完成主干，继续补齐）

### 已完成
- 生命周期标准化（Init/Bootstrap/Run/Teardown）。
- Runtime 事件入口、Tick 入口、渲染入口拆分。
- 状态操作与桥接工厂（bridge_factory / runtime_state_ops）形成分层。
- 运行时错误域/错误码分类与调试面板展示已接入。
- 基础日志、托盘、点击穿透、窗口行为已稳定。

### 待补齐（A-Next）
1. 扩展错误码覆盖范围（目前仍有部分路径保留字符串细节）。
2. 增加“模块级耗时统计 + 可视化面板”（现阶段以错误可视化为主，趋势图仍未补齐）。
3. 增加长稳压测脚本（6h / 12h / 24h 运行）。
4. 统一 Runtime Debug GUI 代码风格并补充 Help/快捷键说明页。

### 验收
- 任一感知/插件异常不导致主循环退出。
- 关键路径问题可定位到模块与输入上下文。

## Phase B：感知能力 1.0（当前主战场）

### B1 屏幕粗感知（已可用）
- DXGI 捕获 + SceneClassifier（MobileCLIP）可用。
- 可输出 scene label + score，并参与任务分类。

### B2 OCR（已打通主链路）
- det + rec ONNX 识别已接通。
- 异步执行、超时降级、last stable 回退已接通。
- OCR summary 去抖与词典归一化已接通。

### B2 待补齐
1. 把 `PP-OCRv5_server_rec_infer.yml` 的字典自动同步到 keys 文件，消除人为错配。
2. 增加 OCR 质量指标面板（平均耗时、置信度分布、丢弃率）。
3. 增加趋势图与 P95 指标可视化，避免当前只看瞬时值。

### B3 系统上下文（已可用）
- process / title / url hint 已进入感知输入。

### B4 融合任务分类（进行中）
- 规则版二级分类已可用（work/game + 子类）。
- 仍需：规则权重可配置化 + 统计准确率。

### 验收
- 核心场景二级分类可持续输出稳定结果。
- 感知线程对主循环影响可观测且受控。

## Phase C：用户感知与动作映射（进行中）

### 已完成
- 摄像头 + 人脸检测 + 情绪分类 + 关键点基本输出。
- 缺帧插值、低光/遮挡降级、平滑策略已落地。

### 待完成
1. 将 head pose / eye open 等特征映射到参数通道（当前映射仍偏基础）。
2. 为映射链路增加阈值配置与可视化调试（校准流程）。
3. 定义“传感器异常 fallback 姿态模板”。

### 验收
- 抖动可控，不出现明显突变。
- 低光遮挡下行为自然退化。

---

## Phase D：插件系统与专家路由（进行中）

### 已完成
- PluginManager + PluginWorker + InferenceAdapter 架构已稳定。
- 超时降频、错误计数、自动禁用与恢复机制已落地。
- 单元/集成测试已覆盖关键异常路径。

### 待完成
1. 真正接入 `.onnx + config.json` 的推理执行路径（当前仍规则主导）。
2. 把 scene/task 的专家路由策略做成可配置表，不硬编码。
3. 打通插件统计上报到调试 UI（超时率、禁用次数、恢复次数）。

### 验收
- 插件异常可完全隔离。
- 在异常压力下主循环稳定。

---

## Phase E：语音与决策中枢（架构完成，能力未完成）

### 已完成
- ASR/Chat 的 Offline/Cloud/Hybrid 抽象层与切换策略框架。
- ASR 已有 VAD 分段 + 运行时指标统计。

### 关键缺口
1. 离线 ASR 未接入真实 whisper.cpp 推理。
2. 云端 ASR 未接入真实 API 请求流程。
3. Chat offline/cloud provider 仍为占位回复。
4. 融合决策仍以规则为主，尚未形成完整 Brain 层策略编排。

### 近期目标
- 先完成 ASR 真模型接入，再完成 Chat 云端真实调用，再收敛策略。

## 7. 风险清单（按优先级）

### P0
1. **占位能力误当可用能力**：ASR/Chat 当前不是最终能力。
2. **配置安全风险**：云端 key 不应写死在代码中。
3. **性能抖动风险**：感知与插件线程在高负载下仍需实测。

### P1
1. OCR 字典不一致导致识别偏差。
2. 任务分类规则硬编码导致场景迁移成本高。

### P2
1. Editor 工程化能力不足导致生产效率受限。
2. Runtime Debug GUI 虽已结构化，但总览趋势图、帮助信息、统一样式仍不足。

## 8. KPI（修正版）

## 近期（4 周）
- 主循环稳定：长跑 6h 无崩溃。
- OCR：超时率下降、稳定摘要输出明显提升。
- 插件：自动禁用/恢复可观测，且不影响主循环。

## 中期（8 周）
- ASR：真实模型接入后可稳定输出文本。
- Chat：至少云端链路可用，具备失败降级。
- 任务分类：核心工作/游戏场景分类稳定可用。

## 9. Definition of Done（当前阶段）

满足以下条件可视为“当前阶段可交付”：

1. 主渲染链路稳定，异常不拖垮主循环。
2. 屏幕 + OCR + 系统上下文 + 摄像头感知可持续运行。
3. 插件异常隔离、降频、禁用/恢复机制可验证。
4. Runtime Debug GUI 具备分区导航、错误降噪、Reminder 搜索/分页/排序、自适应布局等基础可用性。
5. ASR/Chat 至少一条真实能力链路完成替换（不再全占位）。
6. 文档、配置、代码行为一致，新开发者可按文档复现。

---

## 10. 结论

项目已从“单体原型”进入“可演进运行时”阶段：
- 运行时骨架和感知主链路已经成立；
- 插件治理与调试基础具备；
- 下一关键不是“再堆功能”，而是把 **占位能力替换为真实能力**，并用指标驱动稳定性收敛。

本版规划从“愿景文档”转为“执行文档”，后续每周只更新状态与指标，不再重写方向。
